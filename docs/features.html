ADS-B Live Aircraft Tracking

Files: controls/adsb.js (AdsbLiveControl), controls/adsb-labels.js

Polling & Interpolation Loop

Aircraft positions are fetched from the backend proxy every ~5 seconds. Between polls, positions are dead-reckoned (interpolated) every 100ms using each aircraft's heading and ground speed. This produces smooth motion without excessive API calls.

_fetch()
GETs /api/air/adsb/point/{lat}/{lon}/250 using the current user location as centre (or map centre if no GPS). Parses response, updates _lastPositions, queues interpolation.
_interpolate()
Runs every 100ms. Computes dead-reckoned position for each aircraft: newPos = lastPos + (heading_vector × speed × elapsed_ms). Updates the GeoJSON source.

Aircraft Selection Flow

Selection & Tags

_onMapClick(e)
Click on aircraft layer. Finds nearest feature, sets _selectedHex, applies bracket overlay, calls _rebuildTagForHex(hex), updates tracking panel.
_onMapHover(e)
Hover over aircraft. Shows lightweight data tag with callsign, altitude, speed, heading, squawk.
_applySelection()
Updates the bracket layer to highlight _selectedHex. Called after polling to keep bracket on correct aircraft.
_rebuildTagForHex(hex)
Creates a persistent HTML tag marker showing all data for selected aircraft. Updates with each interpolation tick.

Filtering

setTypeFilter(mode: 'all' | 'civil' | 'mil')
Applies a MapLibre filter expression to the ADS-B layer. Military aircraft have a.military = true from the API.
setAllHidden(hidden: boolean)
Hides all aircraft except the tracked one. Used by the HIDE ALL mode in the filter panel.
setHideGroundVehicles(hide: boolean)
Filters ground vehicles from the ADS-B layer.

Trail History

Up to 100 position history points are stored per aircraft hex in _trails. A line layer adsb-trails renders these as a fading track behind each aircraft.

Emergency Squawk Detection

_checkEmergency(hex)
Checks squawk code. If 7500 (hijack), 7600 (comms failure), or 7700 (general emergency), creates an emergency notification via _Notifications.add() and renders a red bracket. Clears notification when squawk returns to normal.

Civil Airports

File: controls/airports.js (AirportsToggleControl)

26 UK and Ireland civil airports are included. Each airport is a GeoJSON feature with ICAO/IATA codes, name, coordinate bounds, and radio frequencies (tower, radar, approach, ATIS).

Markers

Each airport renders an HTML marker showing the ICAO code (lime) and airport name (white). On hover, a frequency panel appears showing all published radio frequencies. On click, the map zooms to the airport's bounding box and the tracking panel shows an airport detail card.

Functions

initLayers()
Creates all HTML markers. Called on map load and on style switch.
_buildFreqPanel(p)
Returns HTML string for the hover frequency popup (tower/radar/approach/atis).
_showAirportPanel(p, coords)
Injects airport detail HTML into the tracking panel and opens it.

Included Airports (sample)

File: controls/raf.js (RAFToggleControl)

24 RAF and USAF bases in the UK. Same marker and detail panel pattern as civil airports but with military styling. Bases include ICAO code, name, and bounding box.

Air-to-Air Refuelling Areas (AARA)

File: controls/aara.js (AARToggleControl)

14 UK AARA polygon zones rendered as filled polygons with dashed outlines. Labels are placed at zone centroids and rotated to align with the longest polygon edge. Label rotation updates in real-time as the map rotates or pitches.

Rendering

Key Functions

_updateLabelRotations()
Recalculates screen-space rotation for each label on map.on('rotate') and map.on('pitch'). Uses computeTextRotate() from MapComponent.

AWACS Orbit Zones

File: controls/awacs.js (AWACSToggleControl)

Approximately 50 AWACS patrol orbit lobes across UK and Atlantic airspace. Each lobe is a 37-point circular polygon. Zones are grouped by region (SW Approaches/Cork, etc.) and rendered as outline layers only.

Range Rings

File: controls/range-rings.js (RangeRingsControl)

Five geodesic range rings at 50, 100, 150, 200, and 250 nautical miles from the user's GPS position. Rings update automatically when location changes.

updateCenter(lng, lat)
Called by setUserLocation() in user-location.js when GPS position updates. Recentres all 5 rings to the new location.
Calls: MapComponent.buildRingsGeoJSON()
initRings()
Creates or recreates the range-rings-lines GeoJSON source and line layer. Called on map load and style switch.

Rings are white dashed lines with distance labels (e.g. "50nm") at the top of each circle.

Notifications Panel

File: notifications/notifications.js

A slide-in panel on the left side of the screen. Notifications are created by the ADS-B system for emergency squawks, landing/departure events, and system status changes.

Persistence

Notifications are saved to localStorage['notifications'] and also synced to POST /api/air/messages. On page reload, notifications are restored from localStorage.

Bell Pulse

When the panel is closed and there are unread notifications, the bell icon in the footer pulses every 15 seconds.

Tracking Panel

File: tracking/tracking.js

A slide-in panel (same side as notifications, tab behavior) that displays the selected aircraft status bar. Also used to show airport and RAF base detail cards.

The ADS-B status bar inside the panel shows: callsign, ICAO hex, registration, aircraft type, altitude, speed, heading, squawk code, and a TRACK/FOLLOW toggle.

Filter Panel

File: air/filter.js

A search panel that opens from the side menu or via Ctrl+F / Cmd+F. Searches in real-time across:

  • Aircraft: callsign, ICAO hex, registration, squawk
  • Airports: ICAO, IATA, name
  • Military bases: ICAO, name

Results are grouped into three sections. Clicking a result zooms to it and opens its detail panel. Notification bell and TRACK button are available per aircraft result.

GPS User Location

File: air/user-location.js

Uses the browser Geolocation API (watchPosition) to track user location. An animated SVG marker shows the GPS position with a pulsing ring, dot, and coordinate typewriter animation.

Location Update Flow

Manual Location Pin

Right-clicking the map shows a context menu with "DROP PIN HERE". Manual pins are persisted in localStorage with a manual: true flag and are never overwritten by GPS updates.

Reverse Geocoding

After every location update (max once per 2 minutes), the coordinates are sent to GET /api/air/geocode/reverse. The returned country name updates the footer location label.

GPS Cache

Last GPS position is cached in localStorage['userLocation'] and expires after 5 minutes. On page load, the cached position is used immediately while a fresh fix is obtained.

Online / Offline Detection

File: map/map.js

The OSM favicon is polled every 2 seconds. On state change, the map style is switched and a footer status pill is updated. All overlays are automatically re-initialized after a style switch via overlay-reinit.js.

Browser window.online / window.offline events are also listened to as supplementary signals, but the OSM poll is the authoritative check.

Clear Overlays

File: controls/clear-overlays.js (ClearOverlaysControl)

A "CLR" button that toggles all overlays off/on. On first press, the current state of every control is snapshotted, then all are turned off. On second press, the snapshot is restored. This allows temporarily clearing all clutter from the map.

_hideAllOverlays()
Saves current state of all controls into savedStates. Calls toggle() on any control that is currently visible to turn it off.
_restoreAllOverlays()
Calls toggle() on controls that were previously visible (according to savedStates) to restore them.
