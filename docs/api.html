Base URL

All API endpoints are served under /api. In development, the backend runs on port 8000. In Docker, the nginx reverse proxy forwards /api to the backend container.

EnvironmentBase URL
Developmenthttp://localhost:8000
Dockerhttp://localhost:8080 (nginx → backend:8000)

Health Check

GET

/health

Returns application health status and current timestamp.

Response

{
  "status": "ok",
  "timestamp": 1710000000
}

ADS-B Proxy

GET

/api/air/adsb/point/{lat}/{lon}/{radius}

Returns live aircraft data within radius nautical miles of (lat, lon). Proxies airplanes.live with a write-through SQLite cache.

Path Parameters

ParameterTypeDescription
latfloatCentre latitude (decimal degrees)
lonfloatCentre longitude (decimal degrees)
radiusintSearch radius in nautical miles (typical: 250)

Response Headers

HeaderValueMeaning
X-CacheHITServed from fresh cache
X-CacheMISSFetched from upstream, now cached
X-CacheSTALEUpstream failed, served stale cache

Response Body

Raw JSON from airplanes.live. Shape:

{
  "ac": [
    {
      "hex": "400f86",
      "type": "adsb_icao",
      "flight": "BAW123  ",
      "r": "G-BNWY",
      "t": "B744",
      "alt_baro": 37000,
      "alt_geom": 37200,
      "gs": 480.5,
      "track": 270.3,
      "lat": 52.4567,
      "lon": -1.2345,
      "squawk": "1234",
      "military": false,
      ...
    }
  ],
  "total": 42,
  "now": 1710000000.123,
  "msg": "No error"
}

Error Responses

StatusWhen
503Upstream unavailable and no stale cache within 30s window

Geocode Proxy

GET

/api/air/geocode/reverse

Reverse-geocodes coordinates to a human-readable address. Proxies Nominatim with a 10-minute SQLite cache (keyed at ~1km resolution).

Query Parameters

ParameterTypeDescription
latfloatLatitude
lonfloatLongitude

Response Body

Raw Nominatim JSON. The frontend extracts address.country for the footer label.

{
  "place_id": 123456,
  "display_name": "Isle of Man, ...",
  "address": {
    "county": "Isle of Man",
    "country": "United Kingdom",
    "country_code": "gb"
  },
  "lat": "54.1453",
  "lon": "-4.4815"
}

Messages (Notifications)

GET

/api/air/messages

Returns all non-dismissed messages, newest first.

Response

[
  {
    "id": 1,
    "msg_id": "emg-400f86-1710000000",
    "type": "emergency",
    "title": "SQUAWK 7700 — BAW123",
    "detail": "General emergency. Alt: 37000ft",
    "ts": 1710000000000,
    "dismissed": false
  }
]
POST

/api/air/messages

201 Created

Create a new message. Idempotent — if msg_id already exists, returns the existing record without creating a duplicate.

Request Body

{
  "msg_id": "emg-400f86-1710000000",
  "type": "emergency",
  "title": "SQUAWK 7700 — BAW123",
  "detail": "General emergency. Alt: 37000ft",
  "ts": 1710000000000
}
DELETE

/api/air/messages/{msg_id}

Soft-delete a single message by its client-generated msg_id. Sets dismissed = true.

Error Responses

StatusWhen
404No message with this msg_id
DELETE

/api/air/messages

Soft-delete all messages (bulk dismiss).

/api/air/tracking

Returns all tracked aircraft, most recently added first.

Response

[

    "follow": true,
    "added_at": 1710000000000
  }

POST

/api/air/tracking

201 Created

Add or update aircraft tracking. If the hex already exists, updates callsign and follow.

  "callsign": "BAW123",

DELETE

/api/air/tracking/{hex}

Remove an aircraft from tracking by its ICAO hex code.

Error Responses

StatusWhen
404No tracked aircraft with this hex

User Settings

Persist and retrieve user preferences. Values are stored as JSON in the user_settings SQLite table, keyed by (namespace, key). All PUT operations are upserts — duplicate keys update the existing row.

Namespaces

NamespaceCurrent keysPurpose
appthemeApplication-wide preferences (e.g. "light" / "dark")
airoverlayStatesAir domain overlay toggle states (JSON object)
space, sea, land, sdr—Reserved for future domain settings
GET

/api/settings

Returns all persisted settings grouped by namespace. Namespaces with no saved settings are omitted.

Response

{
  "app": {
    "theme": "dark"
  },
  "air": {
    "overlayStates": { "airports": true, "military": false }
  }
}
GET

/api/settings/{namespace}

Returns all settings for a single namespace as a flat key/value object. Returns an empty object if the namespace has no saved settings.

Path Parameters

ParameterTypeDescription
namespacestringOne of: app, air, space, sea, land, sdr

Response

{
  "theme": "dark"
}
PUT

/api/settings/{namespace}/{key}

200 OK

Create or update a single setting. If a row for (namespace, key) already exists it is updated in-place; otherwise a new row is inserted.

Path Parameters

ParameterTypeDescription
namespacestringSetting namespace (e.g. app)
keystringSetting key (e.g. theme)

Request Body

{ "value": "dark" }

The value field accepts any JSON-serialisable type: string, boolean, number, object, or array.

Response

{ "status": "ok" }

Cache Headers

The X-Cache response header indicates how the ADS-B proxy served the response:

ValueMeaningData Age
HITServed from fresh SQLite cache< 5 seconds
MISSFetched from airplanes.live, result storedFresh
STALEUpstream unavailable, stale fallback used5–30 seconds old

Error Codes

StatusEndpoint(s)Cause
404DELETE /messages/{msg_id}, DELETE /tracking/{hex}Resource not found
503GET /adsb/point/…Upstream API unavailable and cache expired beyond stale window
422POST endpointsRequest body validation failure (Pydantic)
