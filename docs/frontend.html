Script Load Order

All JavaScript is loaded via <script> tags in index.html. There is no bundler — dependencies are satisfied by load order.

Load order is critical. map.js must load before any control file. air-globals.js must load before any control that references global variables. boot.js must be last to ensure all controls are initialized.

map/map.js — MapComponent

Initializes MapLibre GL, handles connectivity, and provides geometry utilities to all controls.

Connectivity State Machine

Connectivity Detection

_checkInternetConnection() runs every 2s
Polls the OSM favicon URL with a no-cache fetch. On success/failure, calls _switchMapStyle(online) if state changed. Also listens to window.online / window.offline events.
_switchMapStyle(online: boolean)
Sets map.setStyle() to fiord-online.json (online) or fiord.json (PMTiles, offline). All overlay callbacks registered via onStyleLoad are re-fired.
_updateConnStatusPill(online: boolean)
Updates the footer connection indicator text ("ONLINE" / "OFFLINE") and color.

Geometry Helpers

generateGeodesicCircle(lng, lat, radiusNm) → Array<[number,number]>
Generates 181 points forming a great-circle arc at radiusNm nautical miles from the centre. Used by buildRingsGeoJSON and range-rings control.
buildRingsGeoJSON(lng, lat) → {lines, labels}
Builds GeoJSON for 5 geodesic rings at 50, 100, 150, 200, 250nm from the given centre point. Returns separate lines FeatureCollection and labels FeatureCollection (points at ring top with distance text).
Calls: generateGeodesicCircle()
computeCentroid(coordinates) → [number, number]
Shoelace formula to find the area-weighted centroid of a polygon. Used to place AARA zone labels.
computeTextRotate(coordinates) → number
Returns the bearing (degrees) of the longest polygon edge. Used to rotate AARA labels to align with the zone shape.
Calls: computeLongestEdge()
computeLongestEdge(coordinates) → [[lng,lat],[lng,lat]]
Returns the two endpoints of the longest edge of a polygon ring. Used by computeTextRotate.

notifications/notifications.js — _Notifications

Panel Mutex — Notifications vs Tracking

Reusable notification panel shared by all features. Persists to localStorage and syncs with the backend /api/air/messages endpoint.

Public API

MethodDescription
add(opts)Create a notification. opts: {type, title, detail, ts, action?, clickAction?}. Returns generated id.
update(opts)Mutate an existing notification in-place by id.
dismiss(id)Fade out and remove a notification. POSTs DELETE to backend.
clearAll()Dismiss all notifications. POSTs DELETE-all to backend.
toggle()Open/close the panel.
init()Bootstrap: load from localStorage, render, start bell pulse.

Key Behaviors

  • Panel is mutually exclusive with tracking panel (tab behavior).
  • Unread notifications pulse the bell icon every 15 seconds when panel is closed.
  • A "MORE ↓" indicator appears when there are off-screen notifications.
  • Each notification type has a labeled header (e.g. "⚠ EMERGENCY", "✈ FLIGHT").

air/overlay-state.js — Overlay Persistence

Saves all overlay toggle states to localStorage['overlayStates']. Called by each control's toggle() method.

_saveOverlayStates()
Reads the current visible property from every control instance and writes to localStorage. Controls restored from localStorage on next page load.
Reads: roadsControl.roadsVisible, adsbControl.visible, airportsControl.visible, rafControl.visible, aarControl.visible, awacsControl.visible, namesControl.namesVisible, rangeRingsControl.ringsVisible, adsbLabelsControl.labelsVisible

The _overlayStates object is initialized from localStorage['overlayStates'] with defaults if no saved state exists.

air/boot.js — Bootstrap Sequence

Loaded last. On map.once('load'):

  1. Restore 3D pitch from localStorage
  2. Start navigator.geolocation.watchPosition(setUserLocation)
  3. Call _Notifications.init()
  4. Call _Tracking.init()
  5. Call _FilterPanel.init()
  6. Register Ctrl+F / Cmd+F shortcut for filter panel
  7. Run logo typewriter animation IIFE

main.js — Reference Monolith

File: main.js (root, 5176 lines) — not loaded by index.html

The original single-file implementation, kept as a reference during the modular refactor. It is not loaded by the application and has no effect at runtime.

Do not load main.js alongside the modular files — it would re-declare all globals and break the app. Its purpose is purely as a diff reference and fallback.

All functionality in main.js has been split into the modules under frontend/components/. The line-range breakdown of the monolith is documented in the project MEMORY file.

squawk-test.js — Dev Testing Hook

File: squawk-test.js (root, 173 lines) — loaded last in index.html, development only.

Exposes window.sqkTest with helpers for testing emergency squawk detection without needing a real aircraft in distress. It works by monkey-patching window.fetch to return a mock ADS-B response with a controlled squawk code.

Public API (window.sqkTest)

MethodDescription
enterEmergency(squawk)Mock fetch to return one aircraft with the given squawk (7700/7600/7500). Next poll cycle triggers the EMERGENCY notification.
clearEmergency(newSquawk)Mock fetch to return the aircraft with a normal squawk. Next poll triggers SQUAWK CLEARED.
fullFlow(squawk, holdMs)Automated enter → hold → clear sequence. Default hold 8 s. Restores real fetch after one more poll cycle.
allCodes()Fires all three emergency codes (7700 → 7600 → 7500) in sequence, one per poll cycle (~6 s apart). Uses distinct hex IDs so each fires independently.
restore()Restores the real window.fetch and removes test hex entries from adsbControl._prevSquawk.
status()Logs _prevSquawk state table and lists any currently active emergency aircraft from the GeoJSON source.
help()Prints usage summary to the console.
ADS-B polls run automatically every 5 s — after calling enterEmergency() you just wait for the next poll. Always call sqkTest.restore() when done to re-enable real data.
